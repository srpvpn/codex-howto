# Style Guide

Conventions for maintaining the Codex How-To repository.

## Content rules

- prefer official Codex/OpenAI terminology over inferred terminology
- prefer shorter docs that stay accurate over exhaustive docs that drift
- call out unsupported or unverified features explicitly
- use executable paths and commands where possible

## Naming

- numbered lesson folders stay in kebab-case
- top-level reference docs stay uppercase markdown names
- repo examples should prefer Codex path conventions: `AGENTS.md`, `.codex/`, `.agents/`

## Tone

- direct
- technical
- specific
- no marketing fluff
